# WPS租金测算系统 - 后续开发方向规划

> **文档版本**: 1.0  
> **更新日期**: 2026-01-30  
> **状态**: 规划阶段

---

## 一、后续修改方向（代码架构层面）

### 1.1 架构优化

| 优先级 | 方向 | 具体措施 | 预期收益 |
|--------|------|----------|----------|
| 🔴 **高** | 完成依赖注入改造 | 将 `m调息.js`、`m直租.js`、`m测试.js` 等模块的构造函数改为接收 `parameterManager` 参数 | 消除所有隐式全局变量，统一架构风格 |
| 🔴 **高** | 统一错误处理机制 | 建立统一的 `ErrorHandler` 类，所有模块通过依赖注入使用 | 错误处理一致性，便于统一监控和日志记录 |
| 🟡 **中** | 事件驱动架构 | 引入发布-订阅模式，模块间通过事件通信而非直接调用 | 降低耦合度，支持插件化扩展 |
| 🟡 **中** | 模块加载器 | 实现简单的模块加载器，自动管理依赖关系和加载顺序 | 解决手动维护加载顺序的问题 |
| 🟢 **低** | TypeScript 迁移 | 添加 JSDoc 类型注释或逐步迁移到 TypeScript | 编译时类型检查，IDE 智能提示 |

### 1.2 代码质量提升

| 优先级 | 方向 | 具体措施 | 预期收益 |
|--------|------|----------|----------|
| 🔴 **高** | 消除代码重复 | 提取 `m调息.js`、`m直租.js` 中与主模块重复的租金计算逻辑 | 减少维护成本，避免逻辑不一致 |
| 🔴 **高** | 单元测试覆盖 | 为 `clsParameterManager`、`clsCashFlowGenerator` 等核心类编写单元测试 | 提高代码可靠性，支持安全重构 |
| 🟡 **中** | 代码规范统一 | 制定并执行统一的命名规范、注释规范、文件组织规范 | 提高代码可读性，便于团队协作 |
| 🟡 **中** | 性能优化 | 对大数据量场景进行性能分析，优化循环和数组操作 | 提升用户体验，减少计算等待时间 |
| 🟢 **低** | 文档自动化 | 使用 JSDoc 生成 API 文档，与代码同步更新 | 降低文档维护成本 |

### 1.3 配置管理增强

| 优先级 | 方向 | 具体措施 | 预期收益 |
|--------|------|----------|----------|
| 🟡 **中** | 配置验证 | 为所有 CONFIG 对象添加 JSON Schema 验证 | 提前发现配置错误，增强健壮性 |
| 🟡 **中** | 配置持久化 | 支持将配置保存到本地文件或 WPS 文档属性中 | 用户配置不丢失，支持配置导入导出 |
| 🟢 **低** | 配置编辑器 | 提供可视化配置编辑界面 | 降低配置难度，提升用户体验 |

### 1.4 安全性改进

| 优先级 | 方向 | 具体措施 | 预期收益 |
|--------|------|----------|----------|
| 🟡 **中** | 输入验证 | 对所有用户输入参数进行严格的类型和范围验证 | 防止非法输入导致的计算错误 |
| 🟡 **中** | 敏感数据处理 | 如连接外部系统，确保敏感信息（密码等）加密存储 | 符合数据安全规范 |
| 🟢 **低** | 操作审计 | 记录关键操作日志，支持审计追踪 | 满足合规要求，便于问题排查 |

## 二、后续功能完善方向（产品功能层面）

### 2.1 核心功能增强

| 优先级 | 功能 | 描述 | 预期价值 |
|--------|------|------|----------|
| 🔴 **高** | **多方案对比** | 支持同时保存和对比多个测算方案，生成差异分析报告 | 帮助用户选择最优方案 |
| 🔴 **高** | **敏感性分析** | 对关键参数（利率、本金、期限）进行敏感性分析，生成 tornado 图 | 帮助用户识别风险点 |
| 🟡 **中** | **提前还款计算** | 支持计算提前还款后的租金调整方案 | 满足用户灵活还款需求 |
| 🟡 **中** | **逾期/罚息计算** | 支持逾期租金的罚息计算 | 完善风控功能 |
| 🟡 **中** | **增值税计算** | 支持含税和不含税两种计算模式，自动计算增值税 | 满足财税合规需求 |
| 🟢 **低** | **保证金利息** | 支持计算保证金利息收入和冲抵租金 | 完善财务细节 |

### 2.2 业务类型扩展

| 优先级 | 功能 | 描述 | 预期价值 |
|--------|------|------|----------|
| 🔴 **高** | **经营租赁支持** | 区分融资租赁和经营租赁，支持不同的会计处理 | 覆盖更多业务场景 |
| 🟡 **中** | **回租赁业务** | 完善回租赁业务的特殊计算逻辑 | 支持回租业务测算 |
| 🟡 **中** | **联合租赁** | 支持多个出租方的联合租赁测算 | 满足复杂业务需求 |
| 🟢 **低** | **跨境租赁** | 支持多币种、汇率换算的跨境租赁测算 | 支持国际化业务 |

### 2.3 报表与输出

| 优先级 | 功能 | 描述 | 预期价值 |
|--------|------|------|----------|
| 🔴 **高** | **图表可视化** | 生成租金趋势图、本金利息构成图、现金流图等 | 直观展示测算结果 |
| 🟡 **中** | **PDF 导出** | 支持将测算结果导出为 PDF 报告 | 便于分享和存档 |
| 🟡 **中** | **批量生成** | 支持批量读取数据源，生成多个测算表 | 提高工作效率 |
| 🟡 **中** | **自定义模板** | 支持用户自定义报表模板和输出格式 | 满足个性化需求 |
| 🟢 **低** | **邮件发送** | 支持将报告直接发送邮件给指定收件人 | 提升协作效率 |

### 2.4 数据集成

| 优先级 | 功能 | 描述 | 预期价值 |
|--------|------|------|----------|
| 🔴 **高** | **数据库连接** | 支持连接 MySQL/PostgreSQL 等数据库，读取项目信息 | 与企业系统打通 |
| 🟡 **中** | **ERP 集成** | 提供标准接口，支持与主流 ERP 系统集成 | 消除数据孤岛 |
| 🟡 **中** | **LPR 自动更新** | 自动从货币网获取最新 LPR 利率并更新 | 确保利率准确性，减少人工维护 |
| 🟡 **中** | **历史数据归档** | 支持将历史测算数据归档到数据库 | 便于数据分析和审计 |
| 🟢 **低** | **云端同步** | 支持将配置和测算结果同步到云端 | 多设备协同工作 |

### 2.5 用户体验

| 🟢 **低** | **主题切换** | 支持深色/浅色主题切换 | 满足个性化偏好 |

## 三、实施路线图建议

### 第一阶段（1-2个月）：基础夯实
- 完成依赖注入改造
- 统一错误处理机制
- 消除代码重复
- 建立单元测试框架

### 第二阶段（2-3个月）：核心增强
- 多方案对比功能
- 敏感性分析功能
- 图表可视化
- 参数向导

### 第三阶段（3-6个月）：系统集成
- 数据库连接
- ERP 集成接口
- LPR 自动更新完善
- PDF 导出功能

### 第四阶段（6个月以上）：智能化
- AI 智能测算
- 脚本自动化
- 移动端支持
- 云端同步

## 四、决策参考

### 技术债务 vs 功能开发

```
建议比例: 技术债务 : 功能开发 = 4 : 6

原因:
1. 当前系统核心功能已较完善
2. 技术债务（全局变量、代码重复）会影响后续开发效率
3. 良好的基础架构能加速功能开发
```
