### 1.2 核心设计模式

#### 1.2.1 配置驱动架构 (Configuration-Driven Architecture)

**核心思想**: 将配置与业务逻辑分离，通过修改配置而非代码来改变行为。

**代码示例**:
```javascript
// ============== 配置对象定义 ==============
const INITIALIZATION_CONFIG = {
    sections: {
        priceParameters: {
            title: "价格参数填写区域",
            startCell: "A3",
            backgroundColor: "YELLOW",
            parameters: ["Principal", "InterestRate", "Deposit", ...]
        },
        interestRateElements: {
            title: "利率要素填写区域",
            startCell: "F9",
            backgroundColor: "LIGHT_GREEN",
            parameters: ["LPRBenchmarkRate", "FloatingBasisPoints", ...]
        }
        // ... 其他区域配置
    },
    dataValidation: {
        "RateOption": { sourceSheet: "还款设置", sourceRange: "K2:K3" },
        "PaymentInterval": { sourceSheet: "还款设置", sourceRange: "B2:B5" }
        // ... 其他数据有效性配置
    }
};

// ============== 业务逻辑使用配置 ==============
初始化区域(sectionKey) {
    // 从配置获取区域设置
    const sectionConfig = INITIALIZATION_CONFIG.sections[sectionKey];
    
    // 使用配置生成区域
    this.生成区域标题(sectionConfig.title, sectionConfig.startCell);
    
    for (const paramName of sectionConfig.parameters) {
        if (this.是否为公式参数(paramName)) {
            this.设置单元格公式(paramName);
        } else {
            this.设置单元格值(paramName);
        }
    }
}
```

**设计优势**:
1. **可维护性**: 修改配置即可改变行为，无需修改业务代码
2. **可测试性**: 配置可以单独测试和验证
3. **可读性**: 配置集中管理，一目了然
4. **灵活性**: 支持动态扩展新区域或参数

#### 1.2.2 三期模式设计 (Three-Phase Pattern)

**核心思想**: 将租金测算表分为首期、中间期、末期三种类型的公式模板。

**公式模板结构**:
```javascript
// 公式数组结构: arr[公式类型][列索引]
// 公式类型: 1=首期, 2=中间期, 3=末期
// 列索引: 1=期次, 2=支付日, 3=租金, 4=本金, 5=利息...

_createFormulaArray() {
    const arr = [];
    for (let i = 0; i < 5; i++) {
        arr[i] = new Array(13);  // 13列（索引0-12）
    }
    return arr;
}

// 首期公式示例
generateEqualPaymentFormulas() {
    const arr = this._createFormulaArray();
    
    // ===== 首期公式（第1期）=====
    arr[1][1] = "1";  // 期次直接为1
    arr[1][2] = `=EDATE(${params.leaseStartDateCell}, ${params.paymentInterval})`;
    arr[1][3] = `=ROUND(-PMT(${params.interestRateCell}/R11C2,${params.totalPeriodsCell},${params.principalCell},0),2)`;
    arr[1][4] = `=ROUND(-PPMT(${params.interestRateCell}/R11C2,RC[-3],${params.totalPeriodsCell},${params.principalCell},0),2)`;
    // ... 其他列
    
    // ===== 中间期公式（第2期到倒数第2期）=====
    arr[2][1] = "=R[-1]C+1";  // 期次相对引用，前一行的期次+1
    arr[2][2] = "=EDATE(R[-1]C, " + params.paymentInterval + ")";  // 日期相对引用
    arr[2][3] = arr[1][3];  // 租金公式与首期相同
    arr[2][6] = "=RC[-2] + R[-1]C";  // 累积本金相对引用
    // ... 其他列
    
    // ===== 末期公式（最后一期）=====
    arr[3][1] = arr[2][1];  // 期次使用中间期公式
    arr[3][2] = "=EDATE(" + params.leaseStartDateCell + "," + params.paymentInterval + "*" + params.totalPeriodsValue + ")";
    arr[3][4] = `=${params.principalCell} - SUM(R[${-params.totalPeriodsValue + 1}]C:R[-1]C)`;  // 末期本金特殊处理
    // ... 其他列
    
    return arr;
}
```

**为什么使用三期模式？**

1. **首期特殊性**:
   - 期次固定为1，不是相对引用
   - 日期基于放款日计算（EDATE(放款日, 间隔)）
   - 部分公式需要绝对引用参数单元格

2. **中间期通用性**:
   - 期次、日期使用相对引用（R1C1格式），可复制
   - 累积计算基于前一行（R[-1]C）
   - 公式模板一致，适用于第2期到倒数第2期

3. **末期调整性**:
   - 日期需要精确计算到最后一期
   - 本金需要调整确保总和正确（消除舍入误差）
   - 某些汇总公式需要特殊处理

#### 1.2.3 动态属性生成 (Dynamic Property Generation)

**核心思想**: 使用 `Object.defineProperty` 为每个参数动态生成访问属性。

**代码解析**:
```javascript
_createParameterAccessors() {
    const paramNames = this.GetParameterNames();
    
    paramNames.forEach(name => {
        // 动态生成 R1C1 地址属性 (如: PrincipalCellR1C1)
        Object.defineProperty(this, `${name}CellR1C1`, {
            get: () => this.GetConfigValue(name, "CellAddress"),
            configurable: true  // 允许后续重新定义
        });
        
        // 动态生成 A1 地址属性 (如: PrincipalCellA1)
        Object.defineProperty(this, `${name}CellA1`, {
            get: () => this.GetCellAddressA1(name),
            configurable: true
        });
        
        // 动态生成值属性 (如: PrincipalCellValue)
        Object.defineProperty(this, `${name}CellValue`, {
            get: () => this.ReadParameterValue(name),
            configurable: true
        });
    });
}
```

**使用对比**:
```javascript
// 传统方式 - 需要多次调用方法
const cellAddress = p.GetCellAddressA1("Principal");
const value = p.ReadParameterValue("Principal");
const r1c1Address = p.GetConfigValue("Principal", "CellAddress");

// 动态属性方式 - 直接访问属性（更简洁）
const cellAddress = p.PrincipalCellA1;
const value = p.PrincipalCellValue;
const r1c1Address = p.PrincipalCellR1C1;
```

**技术优势**:
1. **语法简洁**: 属性访问比方法调用更简洁
2. **延迟计算**: 属性在访问时才计算，支持动态更新
3. **自动缓存**: getter可以内部实现缓存逻辑
4. **API友好**: 对开发者更直观易用

## 二、高级语法解析

### 2.1 R1C1 引用样式

#### 2.1.1 什么是R1C1样式？

R1C1是一种单元格引用样式，使用**行号(Row)**和**列号(Column)**来定位单元格。

**对比**:
| A1样式 | R1C1样式 | 说明 |
|-------|---------|------|
| B5 | R5C2 | 第5行第2列 |
| D10 | R10C4 | 第10行第4列 |
| A1 | R1C1 | 第1行第1列 |

**相对引用语法**:
| 引用 | 含义 | 示例 |
|-----|------|------|
| R[1]C[1] | 下一行下一列 | 当前单元格右下 |
| R[-1]C | 上一行同列 | 当前单元格正上方 |
| RC[2] | 同行右侧2列 | 当前单元格右边第2个 |
| R[3]C[-1] | 下3行左1列 | 当前单元格左下 |

#### 2.1.2 为什么在租金测算中使用R1C1？

```javascript
// 问题场景: 生成期次公式
// 第1期：期次 = 1
// 第2期：期次 = 第1期 + 1
// 第3期：期次 = 第2期 + 1
// ...

// A1样式的问题：
// 第1期在B26单元格，公式 = 1
// 第2期在B27单元格，公式 = B26 + 1  (硬编码引用)
// 第3期在B28单元格，公式 = B27 + 1  (硬编码引用)
// 缺点: 公式无法复制粘贴到其他位置

// R1C1样式的解决方案：
// 第1期在B26单元格，公式 = 1
// 第2期在B27单元格，公式 = R[-1]C + 1  (相对引用)
// 第3期在B28单元格，公式 = R[-1]C + 1  (相对引用)
// 优点: 所有中间期使用相同公式，可复制粘贴到任何位置

// 代码实现
arr[2][1] = "=R[-1]C+1";  // 中间期期次公式
arr[2][2] = "=EDATE(R[-1]C, " + paymentInterval + ")";  // 中间期日期公式
```

#### 2.1.3 在WPS JSA中使用R1C1

```javascript
// 设置R1C1公式
range.FormulaR1C1 = "=R[-1]C+1";

// 混合使用A1和R1C1
// 引用外部参数使用绝对引用（A1或R1C1绝对格式）
arr[1][3] = `=ROUND(-PMT(${params.interestRateCell}/R11C2,${params.totalPeriodsCell},${params.principalCell},0),2)`;

// 内部相对引用使用R1C1相对格式
arr[2][6] = "=RC[-2] + R[-1]C";  // 累积本金 = 当前行本金 + 前一行累积本金
```

### 2.2 数组批量操作

#### 2.2.1 二维数组操作

```javascript
// 创建二维数组
function _createFormulaArray() {
    const arr = [];
    for (let i = 0; i < 5; i++) {
        arr[i] = new Array(13);  // 创建13列
    }
    return arr;
}

// 公式数组结构
// arr[公式类型][列索引]
// 公式类型: 0未使用, 1=首期, 2=中间期, 3=末期
// 列索引: 0未使用, 1=期次, 2=支付日, 3=租金, ...

// 数据数组转公式数组
arrToArrData(arrFormula) {
    const totalPeriodsValue = this.p.val("TotalPeriods");
    const maxCol = arrFormula[1].length - 1;
    
    // 创建数据数组
    let arrData = [];
    for (let i = 0; i < totalPeriodsValue; i++) {
        arrData[i] = new Array(arrFormula[1].length);
    }
    
    // 填充数据
    for (let row = 0; row < totalPeriodsValue; row++) {
        // 确定使用哪种公式类型
        const rowIndex = row === 0 ? 1 : (row === totalPeriodsValue - 1 ? 3 : 2);
        
        for (let col = 1; col <= maxCol; col++) {
            arrData[row][col - 1] = arrFormula[rowIndex][col];
        }
    }
    
    return arrData;
}
```

## 三、核心算法解析

### 3.1 PMT/PPMT/IPMT 财务函数

#### 3.1.1 等额本息计算

```javascript
// Excel/WPS PMT函数: 计算每期付款额
// PMT(rate, nper, pv, [fv], [type])
// - rate: 每期利率 = 年利率 / 每年还款次数
// - nper: 总期数
// - pv: 现值（本金）
// - fv: 终值（默认为0）
// - type: 0=期末付款(后付), 1=期初付款(先付)

// 代码实现
arr[1][3] = `=ROUND(-PMT(${params.interestRateCell}/R11C2,${params.totalPeriodsCell},${params.principalCell},0),2)`;

// 分解说明:
// ${params.interestRateCell} = 年利率单元格 (如 R5C2 = $B$5)
// R11C2 = 每年还款次数单元格
// ${params.totalPeriodsCell} = 总期数单元格
// ${params.principalCell} = 本金单元格
// 0 = 期末付款
// ROUND(..., 2) = 保留2位小数

// PPMT函数: 计算每期本金
// PPMT(rate, per, nper, pv, [fv], [type])
// - per: 指定期数（第几期）

arr[1][4] = `=ROUND(-PPMT(${params.interestRateCell}/R11C2,RC[-3],${params.totalPeriodsCell},${params.principalCell},0),2)`;
// RC[-3] = 相对引用，当前行向左3列（即期次列）

// 利息 = 租金 - 本金
arr[1][5] = "=RC[-2]-RC[-1]";
```

#### 3.1.2 等额本金计算

```javascript
// 等额本金: 每期本金相同，利息递减

// 每期本金 = 总本金 / 总期数
arr[1][4] = `=ROUND(${params.principalCell}/${params.totalPeriodsCell},2)`;

// 首期利息 = 本金 * 日利率 * 天数
// 日利率 = 年利率 / 360 (金融惯例)
// 天数 = 首期支付日 - 放款日
arr[1][5] = `=ROUND(${params.principalCell}*${params.interestRateCell}/360*(RC[-3]-${params.leaseStartDateCell}),2)`;

// 中间期利息 = 剩余本金 * 日利率 * 天数
arr[2][5] = `=ROUND(R[-1]C[2]*${params.interestRateCell}/360*(RC[-3]-R[-1]C[-3]),2)`;
// R[-1]C[2] = 前一行的剩余本金（向右2列）
```

### 3.3 行插入/删除算法

```javascript
// 插入行算法解析
insertRows(arrData, insertPosition, rowCount, arrFormula, repaymentMethod, options) {
    // 1. 参数验证
    const validation = this.validateInput(arrData, insertPosition, rowCount, 'insert');
    if (!validation.valid) throw new Error(validation.error);
    
    // 2. 计算新的总期数
    const oldTotalPeriods = arrData.length;
    const newTotalPeriods = oldTotalPeriods + rowCount;
    
    // 3. 在指定位置插入空行
    const insertedRows = [];
    for (let i = 0; i < rowCount; i++) {
        insertedRows.push(new Array(arrFormula[1].length));
    }
    arrData.splice(insertPosition, 0, ...insertedRows);
    
    // 4. 判断是否需要重新生成公式模板
    // 等额本息不需要，因为公式不依赖总期数
    // 等额本金需要，因为每期本金 = 总本金 / 总期数
    const needRegenerate = this.shouldRegenerateFormulas(repaymentMethod);
    
    if (needRegenerate) {
        // 临时修改总期数，重新生成公式模板
        const originalTotalPeriods = this.p.val("TotalPeriods");
        this.p.m_worksheet.Range(this.p.addr("TotalPeriods")).Value2 = newTotalPeriods;
        arrFormula = this.generateFormulas(repaymentMethod);
        this.p.m_worksheet.Range(this.p.addr("TotalPeriods")).Value2 = originalTotalPeriods;
    }
    
    // 5. 识别受影响的行
    const modifiedRows = this.identifyAffectedRows(oldTotalPeriods, newTotalPeriods, insertPosition, 'insert');
    
    // 6. 更新受影响行的公式
    for (const rowIndex of modifiedRows) {
        const formulaType = this.getFormulaType(rowIndex, newTotalPeriods);
        arrData = this.updateRowFormulas(arrData, rowIndex, formulaType, arrFormula);
    }
    
    // 7. 更新剩余租金余额公式（涉及汇总范围）
    arrData = this.updateRemainingBalanceFormula(arrData);
    
    // 8. 重新分配本金比例（如果是本金比例还款方式）
    if (repaymentMethod.includes("本金比例")) {
        arrData = this.redistributePrincipalRatio(arrData, newTotalPeriods);
    }
    
    // 9. 更新工作表中的总期数参数
    this.updateTotalPeriods(newTotalPeriods);
    
    // 10. 同步到工作表
    const rng = this.arrDataToDataRange(arrData);
    
    return {
        success: true,
        arrData: arrData,
        oldTotalPeriods: oldTotalPeriods,
        newTotalPeriods: newTotalPeriods,
        modifiedRows: modifiedRows
    };
}
```
