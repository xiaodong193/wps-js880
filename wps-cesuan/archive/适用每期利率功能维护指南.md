## 1. 功能概述

### 1.1 功能说明

**适用每期利率功能**允许在租金测算中使用每期不同的利率，而不是统一使用固定利率($B$5)。

### 1.2 核心价值

- **灵活性**：每期可以设置不同的利率，满足复杂的业务场景
- **精确性**：根据每期的实际利率计算租金、本金、利息
- **自动化**：修改M列利率值后，自动重新计算所有相关数据
- **通用性**：支持所有6种偿还方式

### 1.3 适用场景

1. **利率调整场景**：合同执行期间需要调整利率
2. **分段计息场景**：前期和后期使用不同利率
3. **试算场景**：快速试算不同利率组合的影响
4. **特殊条款场景**：某些期次有特殊的利率约定

### 3.2 核心方法说明

#### 3.2.1 适用每期利率(repaymentMethod)

**位置**：`clsFormulaGenerator`类

**功能**：生成使用每期适用利率的公式

**参数**：
- `repaymentMethod` (string): 偿还方式名称

**返回值**：
- 成功：返回处理后的公式数组
- 失败：返回null

**核心逻辑**：
```javascript
适用每期利率(repaymentMethod) {
    // 1. 调用分发器生成原始公式
    const arrFormula = this.generateFormulas(repaymentMethod);
    
    // 2. 统一替换利率引用
    for (let type = 1; type <= 3; type++) {
        // C列（租金）：${params.interestRateCell} → RC[10]
        if (arrFormula[type][3]) {
            arrFormula[type][3] = arrFormula[type][3].replace(/\$\{params\.interestRateCell\}/g, 'RC[10]');
        }
        
        // D列（本金）：${params.interestRateCell} → RC[9]
        if (arrFormula[type][4]) {
            arrFormula[type][4] = arrFormula[type][4].replace(/\$\{params\.interestRateCell\}/g, 'RC[9]');
        }
        
        // E列（利息）：${params.interestRateCell} → RC[8]
        if (arrFormula[type][5]) {
            arrFormula[type][5] = arrFormula[type][5].replace(/\$\{params\.interestRateCell\}/g, 'RC[8]');
        }
    }
    
    return arrFormula;
}
```

#### 3.2.2 generateFormulas(repaymentMethod)

**位置**：`clsFormulaGenerator`类

**功能**：公式分发器，根据偿还方式调用对应的公式生成方法

**核心逻辑**：
```javascript
generateFormulas(repaymentMethod) {
    // 1. 获取偿还方式配置
    const methodConfig = this.p.config.getRepaymentMethodConfig(repaymentMethod);
    
    // 2. 获取公式方法名称
    const formulaMethod = methodConfig.formulaMethod;
    
    // 3. 动态调用对应的公式生成方法
    return this[formulaMethod]();
}
```

### 3.3 公式替换规则

| 列 | 列索引 | 原引用 | 替换为 | 说明 |
|-----|---------|---------|---------|------|
| 租金 | 3 | `${params.interestRateCell}` | `RC[10]` | 指向M列（每期适用利率） |
| 本金 | 4 | `${params.interestRateCell}` | `RC[9]` | 指向M列（每期适用利率） |
| 利息 | 5 | `${params.interestRateCell}` | `RC[8]` | 指向M列（每期适用利率） |

**智能偏移计算**：
- C列（租金）：目标M列索引12，当前列索引3，偏移 = 12-3 = 9 → **实际使用RC[10]**（向后修正1列）
- D列（本金）：目标M列索引12，当前列索引4，偏移 = 12-4 = 8 → **实际使用RC[9]**（向后修正1列）
- E列（利息）：目标M列索引12，当前列索引5，偏移 = 12-5 = 7 → **实际使用RC[8]**（向后修正1列）

### 3.4 支持的偿还方式

| 偿还方式 | 公式方法 | 涉及列 | RC偏移 |
|---------|---------|--------|--------|
| 等额本息（后付） | generateEqualPaymentFormulas | C, D | RC[10], RC[9] |
| 等额本息（先付） | generateEqualPaymentAdvanceFormulas | E | RC[8] |
| 等额本金（按天计息） | generateEqualPrincipalDailyInterestFormulas | E | RC[8] |
| 等额本金（按期计息） | generateEqualPrincipalPeriodicInterestFormulas | E | RC[8] |
| 本金比例（按期计息） | generatePrincipalRatioPeriodicInterestFormulas | E | RC[8] |
| 本金比例（按天计息） | generatePrincipalRatioDailyInterestFormulas | E | RC[8] |

```

            console.warn(`[${this.MODULE_NAME}] 内存使用增加: ${(memoryIncrease / 1024 / 1024).toFixed(2)}MB`);
        }
        
        return result;
    } catch (error) {
        // 错误处理
    }
}
```

## 6. 常见问题解答

### 6.1 使用相关问题

#### Q1: 如何判断是否应该使用每期适用利率？

**A**: 如果您的业务场景需要每期使用不同的利率，或者需要试算不同利率组合的影响，就应该使用这个功能。如果所有期次都使用相同的利率，使用原有的`createDataRange()`方法即可。

#### Q2: RC[10]、RC[9]、RC[8]是什么意思？

**A**: 
- `RC[10]`：相对引用，从当前行向右偏移10列
  - C列（租金，列索引3）→ M列（每期适用利率，列索引12）：偏移10
- `RC[9]`：相对引用，从当前行向右偏移9列
  - D列（本金，列索引4）→ M列（每期适用利率，列索引12）：偏移9
- `RC[8]`：相对引用，从当前行向右偏移8列
  - E列（利息，列索引5）→ M列（每期适用利率，列索引12）：偏移8

#### Q3: 修改M列利率后，为什么没有自动更新？

**A**: 需要手动触发重新计算：

```javascript
ws.Calculate();  // 触发工作表重新计算
```

或者在公式单元格中添加以下内容：
```javascript
// 在适用每期利率()方法中添加
ws.Range(`${m_COL_RATE_PER_PERIOD}${rentTableStartRow}`).Formula = `=IF(ISBLANK(RC[-1]),${params.interestRateCell},RC[-1])`;
```

#### Q4: 可以混合使用固定利率和每期利率吗？

**A**: 不建议。应该二选一：
- 方案1：使用`createDataRange()`，所有期次使用固定利率($B$5)
- 方案2：使用`使用每期适用利率生成测算表()`，每期使用M列的利率

混合使用会导致公式混乱和计算错误。

#### Q5: 如何恢复到固定利率模式？

**A**: 重新调用原有的方法：

```javascript
const rentalCalc = new clsRentalCalculation();
rentalCalc.Initialize();
rentalCalc.createDataRange();  // 使用固定利率，不使用M列
```

### 6.2 技术问题

#### Q6: 为什么我的公式中没有${params.interestRateCell}？

**A**: 如果使用`适用每期利率()`方法，模板字符串`${params.interestRateCell}`会被替换为RC偏移引用。这是正常行为。如果需要在公式中使用固定利率引用，请使用`createDataRange()`方法。

#### Q7: 正则表达式替换是否会影响其他内容？

**A**: 不会。正则表达式`/\$\{params\.interestRateCell\}/g`只会精确匹配`{params.interestRateCell}`这个字符串模式。其他内容不会被替换。

#### Q8: 如何调试公式生成过程？

**A**: 查看控制台日志，每个步骤都有详细输出：

```javascript
// 控制台输出示例
[clsFormulaGenerator] 公式分发器，偿还方式: 等额本息（后付）
[clsFormulaGenerator] 调用公式生成方法: generateEqualPaymentFormulas
[clsFormulaGenerator] 第1期租金公式已替换利率引用为RC[10]
[clsFormulaGenerator] 第1期本金公式已替换利率引用为RC[9]
[clsFormulaGenerator] 每期适用利率公式生成完成
```

#### Q9: 支持新的偿还方式需要修改什么？

**A**: 只需要在`clsRentalConfig`类的`_initRepaymentMethods()`方法中添加新配置：

```javascript
_initRepaymentMethods() {
    return {
        // ... 现有配置 ...
        
        // 添加新配置示例
        "新偿还方式": {
            formulaMethod: "generateNewMethodFormulas",
            usePrincipalRatio: false,
            needCustomInterval: false,
            convertColumns: [1, 2],
            clearColumns: [],
            headerRange: [1, 10],
            addFrame: ["A:J"],
            extraColumns: []
        }
    };
}
```

并在`clsFormulaGenerator`类中实现对应的方法：
```javascript
generateNewMethodFormulas() {
    // 实现新偿还方式的公式生成逻辑
}
```

#### Q10: 如何处理M列不存在的情况？

**A**: `适用每期利率()`方法假设M列已存在。如果M列不存在，需要先创建：

```javascript
// 在调用使用每期适用利率生成测算表()之前
const rentalCalc = new clsRentalCalculation();
rentalCalc.Initialize();

// 确保M列存在
rentalCalc.确保每期适用利率列存在();

// 然后生成测算表
rentalCalc.使用每期适用利率生成测算表();
```

### 6.3 性能问题

#### Q11: 处理大量期次时性能较慢？

**A**: 可以考虑以下优化：
1. 使用公式缓存（见4.1.1节）
2. 批量设置利率（见4.1.2节）
3. 禁用自动计算，修改完成后统一计算

```javascript
// 禁用自动计算
ws.Calculation = xlCalculationManual;

// 批量操作
// ... 执行批量操作 ...

// 统一计算
ws.Calculation = xlCalculationAutomatic;
ws.Calculate();
```

#### Q12: 内存占用过高？

**A**: 检查是否有内存泄漏：

1. 避免在循环中创建大量对象
2. 及时清理不再需要的引用
3. 使用对象池复用对象

```javascript
// ❌ 不好的做法
for (let i = 0; i < 10000; i++) {
    const obj = new LargeObject();  // 创建大量对象
}

// ✅ 好的做法
const objPool = [];
for (let i = 0; i < 10000; i++) {
    let obj = objPool.pop() || new LargeObject();
    // 使用obj...
    objPool.push(obj);
}
```
