## 📋 前置条件

1. **已安装 WPS Office**（支持JSA宏功能）
2. **WPS宏功能已启用**
   - WPS → 文件 → 选项 → 信任中心 → 启用所有宏
3. **打开正确的工作簿**
   - 包含 `1租金测算表V1` 工作表的文件

## 🚀 快速开始（3步运行测试）

### 第一步：打开宏编辑器
- 按快捷键 `Alt + F11`（Windows）或 `Option + F11`（Mac）
- 或者：开发工具 → JS宏

### 第二步：按顺序加载模块
在宏编辑器中，依次执行以下操作（必须严格按顺序）：

```javascript
// 1. 共享常量模块（最先加载）
// 将 mShared_constants_v2.js 的内容复制粘贴到宏编辑器

// 2. 参数管理器模块
// 将 mParameterManager_v2.js 的内容复制粘贴

// 3. 初始化模块
// 将 mInitialization_v2.js 的内容复制粘贴

// 4. 租金测算模块
// 将 mRentalCalculation_v2.js 的内容复制粘贴

// 5. 现金流量表模块
// 将 mCashFlowGenerator_v2.js 的内容复制粘贴

// 6. 主系统模块
// 将 mMain_v2.js 的内容复制粘贴

// 7. 测试运行器（最后加载）
// 将 m运行测试.js 的内容复制粘贴
```

### 第三步：运行测试
在宏编辑器中执行：

```javascript

// 方法2：完整测试（包含集成测试）

```

## 🔧 故障排查

### 问题1：提示 "Application 未定义"
**原因**：不在WPS环境中运行

**解决**：
- 必须在WPS宏编辑器中运行，不能在浏览器控制台或Node.js中运行

### 问题2：提示 "参数管理器初始化失败"
**原因**：工作表不存在或名称不匹配

**解决**：
- 确保工作簿包含 `1租金测算表V1` 工作表
- 检查工作表名称是否完全一致

### 问题3：提示 "VERSION 未定义"
**原因**：模块加载顺序错误

**解决**：
- 确保 `mShared_constants_v2.js` 是第一个加载的
- 清空宏编辑器，重新按顺序加载

### 问题4：代码有语法错误
**原因**：复制粘贴不完整或有特殊字符

**解决**：
- 重新复制完整文件内容
- 检查是否有中文标点符号
- 查看错误行号，检查对应代码

### 问题5：测试输出看不到
**原因**：控制台被隐藏或未打开

**解决**：
- WPS宏编辑器 → 视图 → 立即窗口
- 在立即窗口执行 `运行快速测试()`

## 🎨 高级用法

### 1. 自定义测试套件

在 `m运行测试.js` 中添加自定义测试：

```javascript
function 运行自定义测试() {
    console.log("\n📦 自定义测试");
    console.log("-".repeat(40));
    
    // 你的测试代码
    var pm = new clsParameterManager();
    pm.Initialize("1租金测算表V1");
    
    console.log("✅ 自定义测试完成");
}
```

### 2. 断言检查

使用单元测试框架的断言：

```javascript
// 引入断言库
var assert = clsAssertions;

// 示例测试
function 测试参数读取() {
    var pm = new clsParameterManager();
    pm.Initialize("1租金测算表V1");
    
    var value = pm.ReadParameterValue("Principal");
    assert.assertNotNull(value, "Principal 值不应为 null");
    assert.assertTrue(value > 0, "Principal 应大于0");
    
    console.log("✅ 参数读取测试通过");
}
```

### 3. 输出到工作表

```javascript
function 测试结果导出() {
    var ws = Application.ActiveSheet;
```

## 📝 开发流程建议

### 开发新功能时
1. 编写功能代码
2. 编写对应的测试函数
3. 运行快速测试验证
4. 运行完整测试确保不破坏其他功能

### 修复Bug时
1. 定位问题代码
2. 编写复现Bug的测试用例
3. 修复代码
4. 运行测试验证修复

### 日常开发
- 每次代码修改后运行 `运行快速测试()`
- 每天结束前运行 `运行所有测试()`
