## ⚠️ VBA to JSA Migration Warning (关键迁移警示 - 核心增强点)
在编写或审查代码时，必须严格遵守以下与 VBA 的**核心差异**，这是导致 Bug 的高发区：

### 1. 属性获取：Value vs Value2
- **VBA 习惯**：`Range("A1").Value`（根据单元格格式自动返回 Date、Currency 等类型）。
- **JSA 规范**：**严禁**滥用 `Value`。默认必须使用 **`Value2`**。
- **原因**：JSA 中 `Value` 处理日期和货币时可能与 JS `Date` 对象类型冲突，导致计算错误。`Value2` 直接返回底层数值，性能更高且类型更安全。
- **例外**：仅当明确需要获取格式化后的文本字符串时，才允许使用 `.Text`。

### 2. 集合与索引：从 1 开始 vs 从 0 开始
- **Excel 对象模型**：`Worksheets(1)`, `Cells(1, 1)` 依然是 **1-based**（与 VBA 一致）。
- **JavaScript 原生数组**：`array[0]` 是 **0-based**。
- **陷阱警示**：切勿将 Excel 对象集合（如 `Range`）直接当作 JS 数组使用 `forEach` 或 `map`（需先转换为数组）。遍历工作表时，注意索引偏移。

### 3. 方括号与圆括号
- **VBA 习惯**：支持参数化属性省略括号，如 `Worksheets 1` 或 `Range("A1") = 100`（不使用 Set 赋值有时省略括号）。
- **JSA 规范**：**所有方法调用必须加括号**。
- **修正**：
  - ❌ `Worksheets 1` (VBA 写法，JSA 报错)
  - ✅ `Worksheets.Item(1)` 或 `Worksheets(1)` (JSA 写法)

### 4. 相等判断
- **VBA 习惯**：`=` 赋值，`=` 判断（无弱类型区分）。
- **JSA 规范**：判断时强制使用 **`===`**（严格相等），赋值使用 **`=`**。
- **陷阱警示**：JS 中 `0 == ""` 为 true，`0 === ""` 为 false。为了避免类型转换 Bug，判断逻辑必须使用 `===` 和 `!==`。

### 5. 逻辑运算符与连接符
- **VBA 习惯**：`And`, `Or`, `Not`；字符串连接用 `&`。
- **JSA 规范**：使用 JS 逻辑符 `&&`, `||`, `!`；字符串连接推荐使用 **模板字符串（反引号 `）** 或 `+` 号。

### 6. 空值与未定义
- **VBA 习惯**：`Nothing`, `Empty`, `Null`。
- **JSA 规范**：对象未赋值为 `null` 或 `undefined`。
- **陷阱警示**：判断对象是否为空时，使用 `if (!obj)` 而非 `If obj Is Nothing`。

## WPS JSA 命名规则与语法规范 (基于 JS 标准增强)

### 1. 类命名
```javascript
class clsParameterManager { }  // 前缀cls表示类
class clsDataValidator { }
```

### 2. 变量声明与命名
- **禁止使用 `Dim` 或 `Set`**：统一使用 `let` 和 `const`。
- **引用类型不需要 `Set`**：直接赋值即可。
```javascript
// ❌ VBA 写法
' Dim rentalRate As Double
' Set targetSheet = Worksheets(1)

// ✅ JSA 写法
const rentalRate = 0.05;          // 常量使用 const
let targetSheet = null;          // 变量使用 let
targetSheet = Worksheets(1);     // 对象赋值不需要 Set，直接 =
this._isInitialized = false;     // 私有变量（下划线前缀）
```

### 3. 常量命名
```javascript
const MODULE_NAME = "租金测算引擎";
const CONFIG_SHEET = "参数配置";
const VERSION = "2.0.0";
```

### 4. 方法/函数命名
```javascript
// 英文函数名（推荐）
calculateNPV() { }
validateInputData() { }

// ⚠️ 逻辑判断：使用 JS 运算符
function checkStatus() {
    // ❌ VBA: If isActive = True And count > 0 Then
    // ✅ JSA:
    if (this.isActive === true && this.count > 0) {
        return true;
    }
}
```

### 5. 属性与数据访问
```javascript
// 公共属性（m_前缀 + camelCase）
this.m_worksheet = null;

// ⚠️ 核心差异：单元格取值
getData() {
    // ❌ 尽量避免：let val = this.m_range.Value; (可能返回 Date 对象)
    // ✅ 推荐：使用 Value2 获取原始数据
    let val = this.m_range.Value2;
    return val;
}
```

### 6. 模块开发规范

**推荐写法（参数注入）：**
```javascript
class clsNewModule {
    /**
     * @param {Object} parameterManager - 参数管理器实例（可选）
     */
    constructor(parameterManager) {
        if (parameterManager && typeof parameterManager === 'object') {
            this.p = parameterManager;
        } else {
            // 回退：自行创建
            this.p = new clsParameterManager();
            this.p.Initialize("默认工作表名");
        }
    }
}
```

### 7. 代码风格规范

- **分号**：每行末尾建议添加分号（虽然 JS 可选，但为了压缩和兼容性建议强制）。
- **标点**：所有符号使用英文半角。
- **引号**：推荐使用反引号 ` ` ` 进行字符串拼接（替代 VBA 的 `&`）。
    ```javascript
    // ❌ VBA 风格
    // let msg = "Hello " + name + ", today is " + date;

    // ✅ JSA 风格
    let msg = `Hello ${name}, today is ${date}`;
    ```
- **注释**：关键逻辑必须注释，特别是涉及 API 差异的地方。
