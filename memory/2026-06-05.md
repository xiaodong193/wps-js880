## 🛠️ 6 种 k() 调用方式

| # | 形式 | 例子 |
|---|------|------|
| 1 | 路径调用 | `k("JSA.getIndexs", 1, 10, 2)` |
| 2 | Lambda | `k("x => x*2", 5)` |
| 3 | $0/$1 索引 | `k("$0 + $1", [10,20])` |
| 4 | f1 / [f1,f3] 列选 | `k("f3", rng)` |
| 5 | 多行代码块 | `k("var s=0; for(...) s+=args[i]; return s;", 1,2,3)` |
| 6 | `...args` 透传 | `k("(...args) => args.join(',')", 1, 2, 3)` |
| + | -r Range | `k("rng => rng.Address()", -r, "A1:B3")` |

支持命名空间：`JSA.` / `Array2D.` / `RngUtils.` / `ShtUtils.` / `DateUtils.` / `IO.` / 全局函数

---

## 💡 关键技术点

- **正则识别路径调用**：`/^\s*([A-Za-z_$][\w$]*(?:\.[A-Za-z_$][\w$]*)+)\s*(\([^)]*\))?\s*$/`
- **缓存机制**：`_lambdaCache` 避免重复解析（第二次调用几乎零开销）
- **多行代码块**：`new Function('args', 'with (JSA) { ' + fn + ' }')` 让用户能写完整 JSA 代码
- **-r 开关**：扫描后续字符串参数，符合地址正则就用 `asRange(a)` 转 Range
- **路径里写死前置参数**：`k("JSA.getIndexs(1, 10)", 2)` 等价于 `k("JSA.getIndexs", 1, 10, 2)`
- **异常容错**：所有路径都 try/catch，失败返回 null（不抛错，对公式极友好）

## 🧠 经验教训

1. **教学 = 一行能跑**：不要把"调用"和"声明"分开讲。`k("JSA.getIndexs", 1, 10, 2)` 这一行就是完整教学。永远让用户先看到"能跑的样子"，再讲原理。
2. **短别名很重要**：`k()` vs `jsaLambda()` —— 单元格公式中字符数是稀缺资源。养成"主名长 + 短别名"的好习惯。
3. **6 种语法分优先级**：路径 > Lambda > $0/$1 > f1 > [f1,f3] > 多行。要确保前面命中后立即 return，否则 lambda 的箭头会被路径误判。
4. **测试要 mock Range**：Node 端没有 WPS 的 Range 对象，要自己写一个 mock（`{ Address: "$A$1" }`）才能测 -r 行为。
5. **try/catch 全包**：在 v4.2.2 之前的实现中，lambda 解析失败会抛错，导致整列单元格都显示 #VALUE!。改成统一返回 null + 控制台 warn，公式容错性大幅提升。
6. **多行代码块要明确 return**：`new Function` 不强制 return，否则会返回 undefined。文档里要明说"必须 return 出去才有结果"。
