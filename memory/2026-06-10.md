## JSA880 z筛选 经验总结

### 1. z筛选 的 _header 自动跳过机制
- `asArray2D()` 会给数据设置 `_header` 属性
- `z筛选` 检测到 `_header` 后，默认自动跳过第1行（skipHeader=1）
- 从 `A2` 开始读数据时，不需要保留表头，直接用 `.z筛选(x => 条件)`
- 从 `A1` 开始读数据时，需要保留表头，必须传第二个参数 `0` 禁用自动跳过：
  ```javascript
  .z筛选((x, i) => i === 0 || 条件, 0)
  ```

### 2. z筛选 两种调用方式
- **Lambda 字符串**：`.z筛选('f7=="是"')` — 简洁，适合简单条件
- **箭头函数**：`.z筛选(x => x.f7 === "是")` — 灵活，适合复杂条件（如 indexOf/includes）
- 两者可混用，但需注意 Lambda 字符串用 `==`，箭头函数用 `===`

### 3. 链式筛选合并
多个条件可合并为一个 z筛选调用，也可链式调用多个 z筛选：
```javascript
// 合并
.z筛选(x => x.f7 === "是" && String(x.f4).includes("宁德"))
// 链式
.z筛选('f7=="是"').z筛选(x => String(x.f4).includes("宁德"))
```

### 4. fN 列号确认
- f1=第1列，f2=第2列，以此类推
- **列号不对是最常见的筛选0行原因**
- 调试方法：遍历第一行打印所有列 `console.log("f" + (i+1) + "=" + JSON.stringify(row[i]))`
