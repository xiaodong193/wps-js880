## 📋 合并单元格逻辑总览

### 基本原则

1. **只在表头区域合并**: 数据行不参与合并
2. **相同值合并**: 相邻的相同值自动合并
3. **标题跨列合并**: 字段标题跨越所有数据列
4. **基于列键分组**: 数据字段标题按列键分组合并

---

## 场景1: 单列字段 + 单数据字段

### 输入数据
```
产品 | 国家 | 月份 | 销量
-----|------|------|-----
A    | 中国 | 1月  | 100
A    | 中国 | 2月  | 200
A    | 美国 | 1月  | 150
```

### 调用方式
```javascript
Array2D.z超级透视(data,
    ['f1', '产品'],      // 行字段
    ['f3', '月份'],      // 列字段（单列）
    ['sum("f4")'],       // 数据字段（单个）
    1                    // 有表头
);
```

### 输出结果
```
        | 1月   | 2月   |
产品    | 销量  | 销量  |
-------|-------|-------|
A      | 300   | 200   |
```

### 合并规则
| 位置 | 合并前 | 合并后 | 说明 |
|------|--------|--------|------|
| 第1行列值 | `1月, 2月` | `1月` (合并1列), `2月` (合并1列) | 每个值只出现1次，无需合并 ✅ |
| 第2行标题 | `月份, 月份` | `月份` (合并2列) | 列字段标题跨所有数据列 ✅ |

**结论**: ✅ 正确（单数据字段时，列值无重复，不合并）

## 场景2: 单列字段 + 多数据字段

### 调用方式
```javascript
Array2D.z超级透视(data,
    ['f1', '产品'],
    ['f3', '月份'],
    ['sum("f4"),average("f4"),count()'],  // 3个数据字段
    1
);
```

### 输出结果
```
        |    1月    |    1月    |    1月    |    2月    |    2月    |    2月    |
产品    |  销量    |  平均    |  计数    |  销量    |  平均    |  计数    |
-------|----------|----------|----------|----------|----------|----------|
A      |   300    |   150    |    2     |   200    |   200    |    1     |
```

### 合并规则
| 位置 | 合并前 | 合并后 | 说明 |
|------|--------|--------|------|
| 第1行列值 | `1月, 1月, 1月, 2月, 2月, 2月` | `1月` (合并3列), `2月` (合并3列) | 每个列值重复 numDataFields 次，需要合并 ✅ |
| 第2行标题 | `月份, 月份, 月份, 月份, 月份, 月份` | `月份` (合并6列) | 列字段标题跨所有数据列 (colKeys.length × numDataFields) ✅ |

**代码逻辑**:
```javascript
// 第1行：列字段值需要合并
var colOffset = baseColIdx;  // = numRowFieldLevels = 1
var numDataFields = 3;       // 3个数据字段
var colKeys = ['1月', '2月'];  // 2个列键

// 循环检测值变化
// ck=0: val='1月', mergeStartCol=1, currentVal='1月'
// ck=1: val='2月', 与前值不同，记录合并: (0, 1, 1, 3) -> 行1列2跨3列
//       然后开始新的合并: mergeStartCol=4, currentVal='2月'
// 循环结束，记录最后合并: (0, 4, 1, 3) -> 行1列5跨3列
```

**结论**: ✅ 正确

## 场景3: 多列字段 + 单数据字段

### 输入数据
```
产品 | 大区   | 省份 | 销量
-----|--------|------|-----
A    | 华东   | 江苏 | 100
A    | 华东   | 浙江 | 200

```

### 调用方式
```javascript
Array2D.z超级透视(data,
    ['f1', '产品'],
    ['f2,f3', '大区,省份'],  // 2个列字段
    ['sum("f4")'],
    1
);
```

### 输出结果
```
        | 大区(标题) | 华东 | 华东 | 华北 | 华北 |
        | 省份(标题) | 江苏 | 浙江 | 北京 | 天津 |
产品    |    销量    | 100  | 200  | 150  | 0    |
-------|------------|------|------|------|------|
A      |            | ...  | ...  | ...  | ...  |
```

### 合并规则
| 位置 | 合并前 | 合并后 | 说明 |
|------|--------|--------|------|
| 第0行标题 | `大区, 大区, 大区, 大区` | `大区` (合并4列) | 第1层级标题跨所有数据列 ✅ |
| 第0行值 | `华东, 华东, 华北, 华北` | `华东` (合并2列), `华北` (合并2列) | 相同大区值合并 ✅ |
| 第1行标题 | `省份, 省份, 省份, 省份` | `省份` (合并4列) | 第2层级标题跨所有数据列 ✅ |
| 第1行值 | `江苏, 浙江, 北京, 天津` | 每个值单独（无重复） | 每个值只出现1次，无需合并 ✅ |

**结论**: ✅ 正确

## 场景4: 多列字段 + 多数据字段

```javascript
Array2D.z超级透视(data,
    ['f1', '产品'],
    ['f2,f3', '大区,省份'],
    ['sum("f4"),average("f4")'],  // 2个数据字段
    1
);
```

### 输出结果
```
        |    大区(标题)    | 华东 | 华东 | 华东 | 华东 | 华北 | 华北 | 华北 | 华北 |
        |    省份(标题)    | 江苏 | 江苏 | 浙江 | 浙江 | 北京 | 北京 | 天津 | 天津 |
        |    销量/平均     | 销量 | 平均 | 销量 | 平均 | 销量 | 平均 | 销量 | 平均 |

```

### 合并规则
| 位置 | 合并逻辑 | 说明 |
|------|----------|------|
| 第0行标题 | `大区` 跨 8 列 | colKeys.length(4) × numDataFields(2) = 8 ✅ |
| 第0行值 | `华东` 跨 4 列, `华北` 跨 4 列 | count × numDataFields = 2 × 2 = 4 ✅ |
| 第1行标题 | `省份` 跨 8 列 | 同上 ✅ |
| 第1行值 | `江苏` 跨 2 列, `浙江` 跨 2 列, ... | count × numDataFields = 1 × 2 = 2 ✅ |
| 第2行标题 | 无合并（每个数据字段独立） | 不合并 ✅ |

**结论**: ✅ 正确

### 合并规则
| 位置 | 合并逻辑 | baseColIdx 计算 |

| 列标题 | `月份` 跨 2 列 | 从第0列开始，跨所有数据列 ✅ |

**结论**: ✅ 正确

## 📊 合并逻辑检查清单

### 单列字段 (numColFieldLevels = 1)

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 列字段值合并 | ✅ | 相同值按 numDataFields 宽度合并 |
| 列字段标题合并 | ✅ | 跨 colKeys.length × numDataFields 列 |
| 行字段列不合并 | ✅ | 行字段列不参与表头合并 |
| baseColIdx 计算 | ✅ | = numRowFieldLevels (默认) 或 0 (hideRowTitles) |

### 多列字段 (numColFieldLevels > 1)

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 每层级标题合并 | ✅ | 每层级的列字段标题跨所有数据列合并 |
| 每层级值合并 | ✅ | 相同值按 (count × numDataFields) 宽度合并 |
| 数据字段标题合并 | ✅ | 最后一行按列键分组合并（每个跨 numDataFields 列） |
| 左上角空白处理 | ✅ | 添加 numRowFieldLevels - 1 个空字符串 |
| hideRowTitles 处理 | ✅ | 不添加空字符串，baseColIdx = 0 |

## 🔍 边界情况检查

### 边界情况1: 空 colKeys
```javascript
var colKeys = [];  // 没有列键
// 预期: 不产生任何合并信息
// 实际: ✅ 正确处理（循环不执行）
```

### 边界情况2: 单个 colKey
```javascript
var colKeys = ['1月'];
var numDataFields = 3;
// 预期: 值 "1月" 跨 3 列合并

```

### 边界情况3: 所有值都相同

var colKeys = ['1月', '1月', '1月'];

## ⚠️ 潜在问题和建议

### 潜在问题1: 非连续相同值不合并

**当前行为**: 只有连续的相同值才会合并
```javascript
// colKeys = ['A', 'B', 'A']
// 结果: A(不合并), B(不合并), A(不合并)
// 而不是: A(跨2列), B
```

**建议**: 这是预期行为，因为数据透视表中列值按排序顺序排列，非连续相同值通常表示数据分组问题。

### 潜在问题2: 合并后样式问题

**当前行为**: 合并单元格后，内容默认居中
**建议**: 可以添加对齐方式设置（如 `mergeAlign: 'left'`）

### 潜在问题3: 大数据量性能

**当前行为**: O(n²) 复杂度（遍历所有 colKeys 查找相同值）
**建议**: 对于大数据量（>10000列），可以考虑优化算法
