# WPS 日志读取功能使用指南

> **版本**: 1.0
> **更新日期**: 2026年2月2日

---

## 📋 功能概述

`jsa_tools.py` 的 `log` 命令提供了多种方式来读取和管理 WPS 宏编辑器立即窗口的日志内容。

---

## 🚀 快速开始

### 方式 1: 粘贴模式（推荐）

```bash
./jsa log --paste
```

**使用步骤**:

1. 在 WPS 宏编辑器立即窗口中运行测试
2. 按 `Cmd+A` (macOS) 或 `Ctrl+A` (Windows) 全选日志
3. 按 `Cmd+C` 或 `Ctrl+C` 复制
4. 在终端中运行 `./jsa log --paste`
5. 粘贴日志内容 (`Cmd+V` 或 `Ctrl+V`)
6. 按 `Ctrl+D` 结束输入

**示例**:

```bash
$ ./jsa log --paste
📋 粘贴模式：请粘贴 WPS 立即窗口的日志内容

使用方法:
1. 在 WPS 宏编辑器立即窗口中按 Cmd+A (macOS) 或 Ctrl+A (Windows)
2. 按 Cmd+C 或 Ctrl+C 复制内容
3. 在下方粘贴日志内容
4. 粘贴完成后按 Ctrl+D (Unix) 或 Ctrl+Z (Windows) 结束输入

============================================================
# 在这里粘贴日志...
[2026-02-02 22:30:15] ✅ 测试12: 组织维度3行1列
[2026-02-02 22:30:15] 结果: 8 行 × 2 列
[2026-02-02 22:30:15] ✅ 测试12完成
^D

============================================================
✅ 已保存 3 行日志到: wps_immediate_log.txt
   使用 --show 查看日志
============================================================
```

### 方式 2: 自动读取（实验性）

```bash
./jsa log
```

此方式会尝试使用 AppleScript (macOS) 或 PowerShell (Windows) 自动读取 WPS 立即窗口内容。

**注意**:
- 需要 WPS 正在运行
- 需要宏编辑器可见
- macOS 需要授予终端辅助功能权限

---

## 📖 命令选项

### 基本命令

| 命令 | 说明 |
|------|------|
| `./jsa log` | 尝试自动读取 WPS 立即窗口 |
| `./jsa log --paste` | 粘贴模式，手动粘贴日志 |
| `./jsa log --show` | 显示已保存的日志 |
| `./jsa log --show --lines 20` | 显示最近 20 行 |
| `./jsa log --clear` | 清空已保存的日志 |
| `./jsa log --analyze` | 分析日志统计 |

### 输出格式

| 选项 | 说明 |
|------|------|
| `--format text` | 纯文本输出（默认） |
| `--format cl` | Claude Code 代码块格式 |

## 🔧 在 JSA 中使用日志工具

### 方法 1: 使用内置日志函数

将 `src/modules/jsa_log_simple.js` 复制到你的 JSA 模块中，然后在测试中使用：

```javascript
// 基本日志
logInfo("开始处理数据");
logWarn("发现空值");
logError("处理失败");
logOk("处理完成");

// 测试块
test("Array2D.z转置", function() {
    var arr = [[1, 2], [3, 4]];
    var result = Array2D.z转置(arr);
    logArray(result, "转置结果");
});

// 运行测试套件
runAllTests();
```

---

### 方法 2: 手动格式化日志

在测试函数中添加清晰的输出：

```javascript
function 测试12_组织3行1列() {
    logSep('=', 60);
    logInfo("========== 测试12: 组织维度3行1列 ==========");

    var arr = 获取测试数据();
    var result = Array2D.z超级透视(
        arr,
        ['f6,f7,f8', '部门,团队,员工'],
        [],
        ['sum("f26")', '销售金额'],
        0,
        1
    );

    logInfo("结果: " + result.z数量() + " 行 × " + result[0].length + " 列");
    logArray(result, "输出预览");
    logOk("✅ 测试12完成");
    logSep('=', 60);
}
```

## 📝 最佳实践

### 1. 使用一致的日志格式

```javascript
// 推荐: 使用统一的前缀
logInfo("[TEST-01] 开始测试");
logWarn("[TEST-01] 数据为空");
logError("[TEST-01] 处理失败");
logOk("[TEST-01] 测试通过");
```

### 2. 添加测试分隔符

```javascript
logSep('=', 60);  // 开始
logSep('-', 40);  // 小节
logSep('=', 60);  // 结束
```

### 3. 记录关键数据

```javascript
logArray(result, "透视结果");
logInfo("数组大小: " + result.z数量() + " 行");
```

## 🔍 日志分析功能

`--analyze` 选项会自动统计：

- ✅ 成功/完成标记
- ⚠️ 警告标记
- ❌ 错误标记
- 总行数
- 问题汇总

**示例输出**:

```
🔍 日志分析:

   总行数: 45
   ✅ 成功: 12
   ⚠️  警告: 2
   ❌ 错误: 0

   发现的问题:
      [2026-02-02 10:15:30] ⚠️  数据为空，跳过处理
      [2026-02-02 10:15:45] ⚠️  字段索引超出范围
```

## ⚠️ 常见问题

### Q: 自动读取失败怎么办？

**A**: 使用粘贴模式：
```bash
./jsa log --paste
```

---

### Q: 如何快速分享日志给 Claude Code？

**A**:
```bash
./jsa log --show --format cl
```
然后复制输出粘贴给 Claude Code。

### Q: 日志保存在哪里？

**A**:
```bash
.temp/wps_immediate_log.txt
```
