# 合并标题维护说明

## 功能概述

本文档说明 `RngUtils.mergeCells` 和 `RngUtils.unMergeCells` 函数的规划、规则和使用方法。

- **mergeCells**: 智能合并透视表标题单元格
- **unMergeCells**: 取消合并并填充值

---

## 一、数据结构基础

### 1.1 前导空白列规则

```
前导空白列数 = max(0, 行字段数 - 列字段数)

┌──────┬──────┬──────┐
│行字段数│列字段数│前导空白│
├──────┼──────┼──────┤
│  1   │  1   │  0   │
│  2   │  1   │  1   │
│  1   │  2   │  0   │
│  2   │  2   │  1   │
│  2   │  3   │  1   │
│  3   │  2   │  1   │
│  3   │  3   │  2   │
└──────┴──────┴──────┘

规律：当 行字段数 > 列字段数 时，需要前导空白列
```

### 1.2 数据结构层级

```
透视表结构（双列字段 + 双行字段为例）:

Row 0: ['', '年份', 年份值, 年份值, ...]      ← 列字段1名 + 列字段1的值
Row 1: ['', '季度', 季度值, 季度值, ...]      ← 列字段2名 + 列字段2的值
Row 2: ['类别', '产品', 数据标题, ...]       ← 行字段名 + 数据标题
Row 3+: ['类别值', '产品值', 数据值, ...]    ← 数据行

标题区域: Row 0 到 Row N-1
数据区域: Row N 开始
```

## 二、mergeCells 函数

### 2.1 函数签名

```javascript
RngUtils.mergeCells = function(rng, strategy, options)
// 别名: RngUtils.z合并单元格

参数:
  rng      - Range对象或地址字符串（如 "A1:D3"）
  strategy - 合并策略
            'cm': 列层级合并（默认，用于透视表标题）
            'c':  按列合并
            'r':  按行合并
            'rm': 先行后列
            undefined: 简单合并整个区域（向后兼容）
  options  - 可选配置对象
            {
              dataRowStart: Number,  // 数据行起始索引
              skipFirstCol: Boolean, // 是否跳过第一列
              skipLastRow: Boolean   // 是否跳过最后一行标题
            }

返回: Range对象（支持链式调用）
```

### 2.2 合并规则（strategy = "cm"）

```
┌──────┬────────────────────────────┐
│ 区域 │        合并规则             │
├──────┼────────────────────────────┤
│列标题│                            │
│ Row0 │ 第2列起，横向连续相同则合并 │
│ Row1 │ 第1列起，横向连续相同则合并 │
│~N-2  │                            │
│ Row  │ 数据标题行，不合并          │
│ N-1  │                            │
├──────┼────────────────────────────┤
│行标题│                            │
│第1列 │ 纵向连续相同则合并（独立）  │
│第2列 │ 纵向连续相同则合并          │
│第3列+│ 纵向连续相同则合并          │
├──────┼────────────────────────────┤
│数据区│ 不合并                      │
│ 域  │                            │
└──────┴────────────────────────────┘

关键点：
- 列标题合并 = 横向合并（跨列）
- 行标题合并 = 纵向合并（跨行）
- 第一列的行标题合并独立判断，不受其他列影响
- 数据标题行永远不合并
- 数据行永远不合并
```

### 2.3 索引说明

```
┌───┬───┬───┬───┬───┐
│ A │ B │ C │ D │ E │←Excel列名
├───┼───┼───┼───┼───┤
│ 0 │ 1 │ 2 │ 3 │ 4 │←0-based索引
├───┼───┼───┼───┼───┤
│ 1 │ 2 │ 3 │ 4 │ 5 │←1-based索引
└───┴───┴───┴───┴───┘

本文档使用 0-based 索引来描述：
- C0:D0 表示 C列到D列（Excel的C1:D1）
- A3:A4 表示 A列到A列的第3-4行
```

## 三、完整示例

### 3.1 例子1：单列字段 + 单行字段（最简单）

#### 数据结构
```javascript
var pivotData = [
    // Row 0: "年份" + 年份值
    ['年份',   '2024', '2024', '2025'],
    // Row 1: "产品" + 数据标题
    ['产品', '销售额', '销售额', '销售额'],
    // Row 2+: 数据行
    ['手机',   100,    120,    105    ],
    ['电脑',   200,    220,    205    ],
];
```

#### 合并前
```
┌────────┬─────────┬─────────┬─────────┬─────────┐
│ 年份   │  2024   │  2024   │  2025   │         │
├────────┼─────────┼─────────┼─────────┼─────────┤
│ 产品   │ 销售额  │ 销售额  │ 销售额  │         │
├────────┼─────────┼─────────┼─────────┼─────────┤
│ 手机   │  100    │  120    │  105    │         │
├────────┼─────────┼─────────┼─────────┼─────────┤
│ 电脑   │  200    │  220    │  205    │         │
└────────┴─────────┴─────────┴─────────┴─────────┘
```

#### 合并后
```
┌────────┬───────────────────┬─────────────────┐
│ 年份   │      2024         │     2025        │
├────────┼─────┬─────┬─────┼─────┬─────┬─────┤
│ 产品   │销售额│销售额│销售额│     │     │     │
├────────┼─────┼─────┼─────┼─────┼─────┼─────┤
│ 手机   │ 100 │ 120 │ 105 │     │     │     │
├────────┼─────┼─────┼─────┼─────┼─────┼─────┤
│ 电脑   │ 200 │ 220 │ 205 │     │     │     │
└────────┴─────┴─────┴─────┴─────┴─────┴─────┘
```

#### 合并规则详解
```
列标题合并（横向）:
  Row 0:
    - A0:A0 → "年份"单独，不合并
    - B0:C0合并 → "2024"连续2次
    - D0:D0 → "2025"单独

  Row 1: 不合并（字段名+数据标题行）

行标题合并（纵向）:
  无合并（每行产品不同）
```

#### 合并规则详解
```
列标题合并（横向）:
  Row 0:
    - A0:A0 → 空
    - B0:B0 → "年份"单独，不合并
    - C0:F0合并 → "2024"连续4次
    - G0:H0合并 → "2025"连续2次

  Row 1:
    - A1:A1 → 空
    - B1:B1 → "季度"单独，不合并
    - C1:D1合并 → "Q1"连续2次
    - E1:F1合并 → "Q2"连续2次
    - G1:G1 → "Q1"单独（不与H1合并，因为H1是"Q2"）
    - H1:H1 → "Q2"单独

  Row 2: 不合并（数据标题行）

行标题合并（纵向）:
  第一列（类别）:
    - A3:A7合并 → "电子产品"连续5次（包含3个手机+2个电脑）
    - A8:A9合并 → "家电产品"连续2次

  第二列（产品）:
    - B3:B5合并 → "手机"连续3次
    - B6:B7合并 → "电脑"连续2次
    - B8:B9合并 → "冰箱"连续2次
```

## 四、unMergeCells 函数

### 4.1 函数签名

```javascript
RngUtils.unMergeCells = function(rng, fillAllRows)
// 别名: RngUtils.z取消合并填充单元格

参数:
  rng         - Range对象或地址字符串
  fillAllRows - Boolean（可选，默认true）
                true:  所有单元格都填充原值
                false: 仅合并区域左上角单元格保留值

返回: undefined
```

### 4.2 使用示例

```javascript
// 当前状态: 已合并的单元格
// B1:D1 合并，值为 "2024"
// E1:F1 合并，值为 "2025"

// 执行 unMergeCells("A1:D3", true)
// 结果: 每个单元格都有原值

// 执行 unMergeCells("A1:D3", false)
// 结果: 仅左上角单元格有值，其余为空
```

```javascript
// 步骤1: 生成透视表
var data = [
```

```javascript
RngUtils.unMergeCells(titleRange, true);
```
