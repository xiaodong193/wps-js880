# superPivot 多层表头修复总结

## 📋 问题概述

superPivot v3.9.0 的多层表头生成存在以下问题：

1. **前导空白列计算错误** - 使用了 `numRowFieldLevels - 1` 而不是 `max(0, numRowFieldLevels - numColFieldLevels)`
2. **单列字段表头行数错误** - 强制使用3行，实际只需要2行
3. **逻辑不一致** - 单列和多列字段使用了不同的处理逻辑

---

## 🎯 修复内容

### 核心修复点

| 修复项 | 修复前 | 修复后 |
|--------|--------|--------|
| 表头行数计算 | `(numColFieldLevels === 1) ? 3 : numColFieldLevels + 1` | `numColFieldLevels + 1` |
| 前导空白列计算 | `numRowFieldLevels - 1` | `max(0, numRowFieldLevels - numColFieldLevels)` |
| 单列字段逻辑 | 单独3行处理 | 统一为2行处理 |

## 📐 正确的表头结构公式

### 表头行数
```
headerRowCount = numColFieldLevels + 1
```

### 前导空白列数
```
leadingBlankCols = max(0, numRowFieldLevels - numColFieldLevels)
```

### 结构示例

| 场景 | 行字段数 | 列字段数 | 表头行数 | 前导空白 |
|------|---------|---------|---------|---------|
| 单列+单行 | 1 | 1 | 2 | 0 |
| 双列+单行 | 1 | 2 | 3 | 0 |
| 单列+双行 | 2 | 1 | 2 | 1 |
| 双列+双行 | 2 | 2 | 3 | 0 |
| 三列+双行 | 2 | 3 | 4 | 0 |
| 双列+三行 | 3 | 2 | 3 | 1 |
