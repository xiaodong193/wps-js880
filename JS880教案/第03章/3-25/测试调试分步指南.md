## 🎯 准备工作

### 步骤 1: 打开文件

#### 1.1 打开 Excel 文件
1. 打开 WPS Office 或 Microsoft Excel

#### 1.2 打开 VBA 编辑器

- 按 `Alt + F11`

- 或通过菜单：工具 → 宏 → Visual Basic 编辑器

#### 1.3 检查环境

- 左侧应该看到"项目"面板

- 应该看到"模块"文件夹

## 🐛 调试问题

### 常见问题 1: Array2D 未定义

**错误信息**：
```
运行时错误 '424':
对象必需
```

**原因**：
JSA880.js 没有正确加载

**解决方法**：
1. 返回[步骤 2](#步骤-2-加载-jsa880js如果还没有)
2. 确认 JSA880.js 代码已粘贴到模块中
3. 检查模块名称是否正确
4. 重新运行测试

---

### 常见问题 2: 测试数据为空

**错误信息**：
```
结果行数: 0
❌ [FAIL] 基础透视表
```

**原因**：
测试数据生成函数可能有问题

**解决方法**：
1. 在立即窗口中测试数据生成：
   ```javascript
   var data = createTestData().salesData
   ? data.length
   ```
2. 应该输出大于 0 的数字
3. 如果是 0，检查 `createTestData()` 函数是否正确

---

### 常见问题 3: superPivot 函数报错

**错误信息**：
```
运行时错误 '1004':
应用程序定义或对象定义错误
```

**原因**：
可能是函数参数不正确或数据格式问题

**解决方法**：
1. 打开 SuperPivot 函数源码（在 JSA880.js 中）
2. 在函数开头添加调试输出：
   ```javascript
   Console.log("数据行数: " + arr.length);
   Console.log("行字段: " + JSON.stringify(rowFields));
   ```
3. 重新运行测试
4. 查看输出，定位问题

---

### 常见问题 4: 测试超时或卡死

**现象**：
测试运行很长时间没有响应

**解决方法**：
1. 按 `Esc` 或 `Ctrl + Break` 停止测试
2. 修改测试，只运行单个测试：
   ```javascript
   BasicTests.test1_1()
   ```
3. 检查数据量是否过大
4. 检查是否有死循环
